# Next Word Predictor — Free Tier Render Deploy

## How it works

Training is split from serving so the server never needs TensorFlow:

```
Your Machine                        Render (Free Tier)
────────────────                    ──────────────────
train.py                            app.py
  │ TensorFlow + tf2onnx              │ onnxruntime only (~50MB)
  │ trains LSTM                       │ loads model.onnx
  │ exports ──────────────────────────► model.onnx
  │         ──────────────────────────► tokenizer.pkl
  └─ commit both files to GitHub      └─ serves /predict
```

## Step 1 — Train locally

```bash
pip install tensorflow tf2onnx onnxruntime

python train.py
# Paste your text, press Enter twice
# Produces: model.onnx  tokenizer.pkl
```

## Step 2 — Commit everything to GitHub

```
your-repo/
├── app.py
├── requirements.txt
├── render.yaml
├── model.onnx        ← generated by train.py
└── tokenizer.pkl     ← generated by train.py
```

```bash
git add app.py requirements.txt render.yaml model.onnx tokenizer.pkl
git commit -m "add trained model"
git push
```

## Step 3 — Deploy on Render

1. Go to https://render.com → **New → Web Service**
2. Connect your GitHub repo
3. Render detects `render.yaml` automatically — click **Deploy**
4. Done! Your service will be live at `https://your-app.onrender.com`

## Usage

```bash
curl -X POST https://your-app.onrender.com/predict \
  -H "Content-Type: application/json" \
  -d '{"seed_text": "the cat"}'

# → {"seed": "the cat", "next_word": "sat"}
```

Or open `https://your-app.onrender.com/docs` for the interactive UI.

## Retrain on new text

Just run `train.py` again, then `git push`. Render redeploys automatically.
